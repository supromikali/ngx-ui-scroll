All the datasources listed above are valid. The first version (datasourceCallback) introduces simplest callback-based signature when we need to pass third function argument and call it explicitly with the items array argument. We assume that this.getData(index, count) returns an appropriate part of the data set we want to virtualize, which is an array of count items started from index position.

datasourcePromise and datasourcePromise2 introduce promise-based signature when an object of Promise type needs to be returned. Resolving this promise is our responsibility, and we assume that this.getDataPromise method resolves necessary part of the data set based on index and count parameters.

datasourceObservable and datasourceObservable2 introduce observable-based signature when an object of Observable type needs to be returned. Similar to promise case we need to be sure that the next method is being called in appropriate time, and we assume that this.getDataObservable method that delivers necessary part of the data set based on index and count parameters.
